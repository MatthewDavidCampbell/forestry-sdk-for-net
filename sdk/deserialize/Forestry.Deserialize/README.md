# Forestry Deserialize
Both small and heavy streams of different media (i.e. XML || JSON || custom) flow into Forestry and these become after validation subject to analysis.  Media is streamed into a neutral format subject to queries that feed user reports.  A common problem is pulling only fractions of large data sets or processing only reoccuring sections without loading the entire document into memory.  Streams work best with pipeline processing and the enumeration over values with custom dimensions rather than an entire data structure.
